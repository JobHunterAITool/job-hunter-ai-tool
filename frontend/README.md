# Job Hunter AI Tool -- Frontend

This repository contains the React + Vite frontend for the **Job Hunter
AI Tool** capstone project.

The frontend allows users to enter job search information, upload a
resume, send requests to the backend API, and view ranked job matches
returned by the system.

The goal of this interface is to provide a simple way for users to
submit job search information and quickly see relevant job results
generated by the backend machine learning service.

# Overview

The frontend collects user input through a search form and sends that
information to the backend API. The backend processes the request and
returns ranked job matches, which are then displayed in the interface.

Users may also upload a resume file (PDF or DOCX). The resume is sent to
the backend where text is extracted and used to assist with job
matching.

The application was created using:

-   **React** for the component-based interface
-   **Vite** for fast development builds and hot module replacement
-   **Axios** for communication between the frontend and backend API

# Running the Development Server

Start the development server with:

npm run dev

After starting the server, Vite will provide a local address, typically:

http://localhost:5173

The development server supports hot module replacement, meaning the page
will update automatically when changes are made to the code.

# Backend API Connection

The frontend communicates with the backend using **Axios**.

API requests are defined in:

src/services/api.js

The frontend sends requests to the backend for:

-   Job search requests (`POST /search`)
-   Resume uploads (`POST /upload-resume`)

The backend URL can be configured using an environment variable:

VITE_API_BASE_URL=http://127.0.0.1:8000

If no environment variable is set, the frontend defaults to:

http://127.0.0.1:8000


# Resume Upload Feature

Users may upload a resume by dragging and dropping a file into the
upload area or selecting a file manually.

Supported formats:

-   PDF
-   DOCX

The resume file is sent to the backend where a short text preview is
extracted and returned to the frontend.

## Frontend Testing

The frontend includes automated tests for key UI components using **Vitest** and **React Testing Library**. These tests simulate user interactions, mocked API responses, loading states, error states, and rendered output without requiring the backend server to be running.

The current test suite covers:

- `ResumeUpload.jsx`
- `SearchForm.jsx`
- `JobResults.jsx`

### Test Coverage

The `ResumeUpload` component tests verify that:

- The component renders correctly
- An error message appears when uploading without selecting a file
- Valid **PDF** and **DOCX** files are accepted
- Invalid file types are rejected
- Successful uploads display returned resume preview text
- Upload failures display an appropriate error message

The `SearchForm` component tests verify that:

- The search form renders correctly
- Input fields update when a user types
- Search requests send the correct payload to the API
- Old results and errors are cleared before a new search starts
- Inputs and the submit button are disabled while loading
- Search errors are handled correctly

The `JobResults` component tests verify that:

- The ranked results section renders correctly
- The loading state appears during a search
- Error messages are displayed when a request fails
- The initial empty state appears before a search
- The no-results message appears after a completed search with no matches
- Job result cards display title, company, location, score, and matched skills
- Jobs with no matched skills display `None`

### Running Tests

Navigate to the frontend directory and run:

```bash
cd frontend
npm install
npm run test
```

Vitest runs in watch mode by default and automatically reruns tests when files change.

To run the test suite once and exit:

```bash
npm run test -- --run
```
